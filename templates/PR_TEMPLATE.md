Use this structure for every pull request. When HOCA runs `create-pr.sh`, each section below is filled from the task run directory (`task-spec.json` for **Task Spec**, git log/diff stat for **Changes**, `tests-summary.md` or structured validation reports for **Validation**, review and manager decision artifacts for **HOCA Review Notes** and **Code Review**, `risk-notes.txt` for **Risk**, run metadata for **Run Context**, and the optional `--issue-id` for **Linked Issue**). The PR body always ends with an explicit **Auto-merge** line (default: disabled). For manually written PRs, keep **Validation** to concrete commands and outcomes, and use **Risk** for limitations, rollout impact, or anything reviewers should treat carefully.
